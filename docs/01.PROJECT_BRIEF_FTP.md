# 01.PROJECT_BRIEF.md

Unified Project Brief (FTP Deployment Version)

## 1. Project Overview

This project is a **cosmetics e-commerce website** with a hybrid
architecture.

The website is divided into two major parts:

1.  **Public-facing website**
    -   main page
    -   landing pages
    -   brand / product introduction pages
    -   sub pages
    -   built with **HTML + CSS + Vanilla JavaScript** or optional **PHP
        includes**
2.  **Protected application area**
    -   login / signup
    -   user account features
    -   admin dashboard
    -   built with **Next.js**

This structure is intentionally chosen to balance:

-   strong SEO performance
-   fast public-page loading
-   maintainable front-end architecture
-   scalable app/dashboard development

------------------------------------------------------------------------

## 7. Deployment Strategy (FTP Version)

### 7.1 Public Website

Deployment method: - develop locally - upload static files through
**FTP** - optional PHP template structure on the hosting server

Recommended server structure:

``` text
/public_html
  index.html or index.php
  /pages
  /assets
  /includes
```

Recommended tools: - FileZilla - WinSCP - VS Code FTP extension -
EditPlus / direct file editing workflow if needed

### 7.2 Next.js App / Dashboard

Important: A traditional shared-hosting FTP environment may **not**
support a Next.js runtime.

So one of the following must be chosen:

#### Option A

Deploy only the **public website** to FTP hosting, and host the Next.js
dashboard separately on: - Vercel - separate VPS - Node.js hosting

#### Option B

If the server supports Node.js runtime and process management: - deploy
the Next.js app separately on the same infrastructure - do not treat
Next.js like a simple FTP-upload static site unless using static export
only

Recommended practical setup: - public pages on FTP hosting -
app/dashboard on a separate modern Node hosting platform

## 8. SEO Strategy

SEO applies mainly to the **public website layer**.

Required rules: - semantic HTML - one H1 per page - clear H2/H3
structure - unique title and meta description - canonical tags - Open
Graph tags - sitemap.xml - robots.txt - JSON-LD schema where appropriate

Content priorities: - homepage: brand + core category intent - product /
service pages: long-tail intent keywords - blog / insights:
informational discovery traffic - FAQ: structured search-friendly
content

## 11. Key Constraint Clarification

The provided PRD note correctly highlights that deployment environment
matters.

For the **FTP version**: - static public pages are suitable - PHP
includes are optional and practical - Next.js requires separate runtime
support unless exported statically - admin/dashboard must not be assumed
to work on simple shared FTP hosting without Node support

## 12. AI Development Instructions

AI must understand this project as a **hybrid website system**:

-   Public SEO pages = HTML/CSS/JS or PHP
-   Auth/admin/dashboard = Next.js

AI must not: - force all pages into Next.js - assume shared FTP hosting
can run server-rendered Next.js automatically - overcomplicate public
SEO pages with unnecessary frameworks

AI should: - prioritize SEO on public pages - keep design consistent
across both layers - reuse shared design tokens and components - clearly
separate public layer and app layer

# 1. Project Description

The project is a **cosmetics e‑commerce website** built with a hybrid
architecture.

The system is divided into two major layers:

1.  Public Website (SEO‑focused)
2.  Application Layer (authentication and management)

Public pages must prioritize:

-   SEO
-   fast loading
-   simple structure
-   clear brand presentation

Protected pages must support:

-   login
-   signup
-   user account features
-   admin dashboard
-   operational management

------------------------------------------------------------------------

# 2. Public Website Requirements

Public pages include:

-   Home
-   Product / Service introduction pages
-   Brand / About pages
-   Landing pages
-   Blog / insights (optional)
-   FAQ
-   Contact
-   Policy pages

Public pages must be implemented using:

HTML5 (Semantic) CSS3 Vanilla JavaScript

Optional: PHP includes for shared header/footer templates.

Reasons:

-   best SEO performance
-   fastest loading speed
-   simple hosting compatibility
-   easy maintenance

# 5. Deployment Environments

The system must support two deployment models.

## Model A --- FTP Hosting

Public pages: uploaded via FTP to traditional hosting.

Application layer: hosted separately on Node‑compatible infrastructure
(Vercel, VPS, etc.).

Important: Shared FTP hosting often cannot run server‑side Next.js.

## Model B --- Cloudflare Architecture

Public pages: deployed on Cloudflare Pages.

Backend / APIs: Cloudflare Workers or external backend.

Authentication and data operations: handled through APIs or external
services.

Important: Cloudflare Pages is primarily static hosting. Full Node.js
runtime support must be verified before using Next.js server features.

------------------------------------------------------------------------

# 6. SEO Requirements

SEO applies mainly to public pages.

Rules:

-   semantic HTML
-   one H1 per page
-   proper H2/H3 structure
-   unique page titles
-   meta descriptions
-   canonical tags
-   Open Graph tags
-   structured data (JSON‑LD)
-   sitemap.xml
-   robots.txt

Images must:

-   use descriptive alt text
-   use optimized formats (WebP)
-   use responsive sizes

# 8. Development Process (SDLC)

Phase 1 --- Requirement Analysis Define features, pages, deployment
environment.

Phase 2 --- Planning Create sitemap, component structure, design system.

Phase 3 --- Design Define layout patterns and reusable components.

Phase 4 --- Implementation Build public site and application layer
separately.

Phase 5 --- Testing Responsive testing, SEO audit, accessibility review.

Phase 6 --- Deployment Deploy according to chosen environment (FTP or
Cloudflare).

Phase 7 --- Maintenance Content updates, SEO improvement, dashboard
enhancement.

Use these documents as the source of truth.

Important environment constraints:

- This project is deployed via FTP to a traditional hosting server.
- Public-facing pages must be built using HTML, CSS, and minimal Vanilla JavaScript.
- PHP includes may be used for shared components such as header and footer.
- Do NOT convert the public website into a framework-based SPA.

Architecture rules:

Public website layer:
- HTML
- CSS
- minimal JavaScript
- SEO optimized
- fast loading

Application layer (separate system):
- login
- signup
- admin dashboard
- user account features
- may use Next.js or other frameworks
- must not break the public website structure

When generating new pages:

1. Select the correct template from PAGE_TEMPLATES.md
2. Reuse components from COMPONENT_LIBRARY.md
3. Follow DESIGN_SYSTEM.md
4. Respect SEO_RULES.md
5. Keep JavaScript minimal

Always prioritize:

- SEO
- semantic HTML
- fast loading
- maintainable code

Use these documents as the source of truth.

Important environment constraints:

- The public website is deployed on Cloudflare Pages.
- Public pages must remain static and SEO-friendly.
- Avoid server-side frameworks for public content.

Architecture rules:

Public website layer:
- static HTML
- CSS
- minimal JavaScript
- optimized for CDN delivery

Protected application layer:
- login
- signup
- dashboard
- admin tools

These may be implemented using:

- Cloudflare Workers
- Hono APIs
- external backend services
- or a separately hosted Next.js application

Do NOT assume that full server-side Next.js features are available within Cloudflare Pages without the proper runtime configuration.

When generating pages:

1. Choose the appropriate template from PAGE_TEMPLATES.md
2. Reuse UI from COMPONENT_LIBRARY.md
3. Follow DESIGN_SYSTEM.md
4. Respect SEO_RULES.md
5. Optimize for static delivery and CDN performance

Priorities:

- SEO-first structure
- semantic HTML
- minimal JavaScript
- high performance

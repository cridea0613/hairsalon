# COMPONENT_LIBRARY.md

Reusable UI Component Standards for Vibe Coding Projects

This document defines the **standard UI components** used across the
public website. AI tools must reuse these components instead of
generating new inconsistent UI patterns.

Purpose: - maintain design consistency - accelerate AI-assisted
development - avoid duplicate UI implementations - keep CSS predictable
and modular

------------------------------------------------------------------------

# 1. Navigation Header

Purpose: Primary site navigation and brand visibility.

Structure:

```{=html}
<header class="site-header">
```
::: {.container .nav-container}
    <div class="logo">Brand</div>
    <nav class="main-nav">
      <a href="/">Home</a>
      <a href="/service">Service</a>
      <a href="/about">About</a>
      <a href="/contact">Contact</a>
    </nav>
:::

```{=html}
</header>
```
Rules: - Sticky optional - Navigation must be accessible - Mobile menu
toggle required

------------------------------------------------------------------------

# 2. Hero Section

Purpose: First visual section of the homepage that communicates the core
value.

Structure:

```{=html}
<section class="hero">
```
::: {.container .hero-content}
    <h1>Main Headline</h1>
    <p class="hero-subtitle">Short supporting description.</p>
    <a class="btn-primary" href="#">Get Started</a>
:::

```{=html}
</section>
```
Design Guidelines: - large headline - strong CTA - clear visual
hierarchy

------------------------------------------------------------------------

# 11. Component Rules for AI

AI must:

1.  reuse existing components
2.  avoid generating duplicate UI patterns
3.  follow defined class naming
4.  maintain consistent spacing
5.  keep components modular

AI should **extend components, not reinvent them**.

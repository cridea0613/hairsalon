# Vibe Coding Public Website Starter Documentation

## 1. Project Overview

This project builds a **public-facing website optimized for SEO, speed,
and clarity**.

Primary goals: - SEO‑friendly structure - High performance - Clean
maintainable code - AI‑assisted development compatibility

The website is **content-driven**, not a complex web application.

Typical pages: - Landing pages - Company introduction - Service pages -
Blog articles - Contact pages

------------------------------------------------------------------------

## 2. Technology Stack

Primary stack:

-   HTML5 (Semantic)
-   CSS3 (Flexbox + Grid)
-   Vanilla JavaScript (ES6+)

Optional backend support:

-   PHP (for template includes)

Why this stack:

-   Excellent SEO compatibility
-   Fast loading
-   Simple deployment
-   Easy maintenance

------------------------------------------------------------------------

## 3. Performance Targets

Target Lighthouse scores:

-   Performance: 90+
-   Accessibility: 95+
-   SEO: 95+
-   Best Practices: 90+

Performance rules:

-   minimal JavaScript
-   optimized images
-   mobile‑first design
-   avoid heavy libraries

------------------------------------------------------------------------

## 4. Website Structure

Typical page structure:

-   Home
-   Service / Product
-   About
-   Blog
-   Contact
-   Landing pages

Homepage functions as the main navigation hub.

Blog pages expand SEO reach. Landing pages support marketing campaigns.

## 6. HTML Development Rules

Use semantic HTML.

Preferred tags:

header nav main section article footer

Heading rules:

-   One H1 per page
-   H2 for sections
-   H3 for subsections

Accessibility:

-   descriptive alt text
-   real button elements
-   keyboard navigation
-   visible focus states

------------------------------------------------------------------------

## 7. CSS Design System

Use CSS variables.

Example:

:root { --color-primary: #007bff; --color-text: #1a1a1a; --space-sm:
8px; --space-md: 16px; --space-lg: 32px; --radius-md: 12px; }

Spacing scale:

4px 8px 16px 24px 32px 48px

Typography:

Inter, Pretendard, system-ui, sans-serif

Line-height: \~1.6

------------------------------------------------------------------------

## 8. UI Design Philosophy

Style: Modern Minimalism

Characteristics:

-   generous whitespace
-   strong typography hierarchy
-   simple grid layout
-   subtle shadows
-   soft borders

Avoid:

-   heavy animations
-   cluttered layouts

------------------------------------------------------------------------

## 9. JavaScript Guidelines

Use JS only for:

-   navigation toggle
-   sticky header
-   FAQ accordion
-   smooth scroll
-   small UI interactions

Avoid:

-   large frameworks
-   heavy client-side logic

------------------------------------------------------------------------

## 10. SEO Strategy

Each page must include:

-   unique title
-   meta description
-   heading hierarchy
-   semantic structure

Additional SEO:

-   canonical tag
-   Open Graph tags
-   JSON-LD schema
-   sitemap.xml
-   robots.txt

Image SEO:

-   descriptive alt text
-   optimized sizes
-   WebP format

------------------------------------------------------------------------

## 11. Accessibility Standards

Follow WCAG guidelines.

Requirements:

-   contrast ratio ≥ 4.5:1
-   keyboard navigation
-   visible focus indicators
-   ARIA labels when necessary

------------------------------------------------------------------------

## 12. AI-Assisted Development Workflow

Phase 1 --- Planning\
Define page structure and content hierarchy.

Phase 2 --- HTML Structure\
Generate semantic HTML layout.

Phase 3 --- CSS Styling\
Apply modular design system styles.

Phase 4 --- JavaScript Enhancements\
Add lightweight UI interactions.

Phase 5 --- Audit\
Check SEO, accessibility, and performance.

------------------------------------------------------------------------

## 13. Coding Style Rules

Naming guidelines:

.hero-section .nav-container .feature-card .cta-button

Principles:

-   reuse components
-   avoid duplicate styles
-   keep code readable
-   prioritize simplicity

------------------------------------------------------------------------

## 14. Instructions for AI

When AI generates code:

1.  respect the folder structure
2.  maintain semantic HTML
3.  avoid unnecessary libraries
4.  follow CSS design system
5.  prioritize SEO and accessibility

AI must produce **clean, maintainable production-ready code**.
